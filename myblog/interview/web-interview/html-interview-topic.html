HTML

语义化就是用合理正确的标签来展示内容，比如用h1~h6标签定义标题。

【好处】

  • 更方便用户阅读，css样式丢失时可以使页面呈现清晰的结构。
  • 有利于SEO搜索引擎优化[搜索引擎会根据标签来确定上下文和各个关键字的权重]。
  • 有利于其他设备如盲人阅读器对网页的解读,可以根据语义渲染网页。
  • 有利于开发维护，代码更具有可读性，和css3关系更和谐。

【link放在head中】

把link标签放在head之间时规范要求的内容。此外，这种做法可以让页面逐步呈现，提高了用户体验。
如果将css样式表放在文档底部附近，会使许多浏览器(包括IE)不能逐步呈现页面。
浏览器会阻止渲染页面的原因是防止页面样式发生变化时，重绘页面中的元素。
link标签放在head之间这种做法可以防止呈现给用户空白页面或无样式的内容。

【script放在/body前】

这是由于脚本在下载和执行期间会阻止HTML解析。将script放在底部，可以保证html首先完成解析，将页面尽早呈现给用户。
但当js脚本中使用了document.write()方法来创建当前的文档内容时，就需要将它放置在head中，避免页面完成后执行js时遇到document.write，进行重绘。
同时，在文档(document)解析完之前，浏览器无法下载脚本。
在IE中可以解决这个问题，给script标签设置defer属性(defer="defer")，放置在head中。defer属性规定是否对【脚本执行】进行延迟，直到页面加载完成为止。但只在IE中有效。

渐进式渲染是用于提高网页性能(尤其是提高用户感知的加载速度)，以尽快呈现页面的技术。
在以前互联网宽带较小的时期，这种技术更为普遍。而如今，移动终端的盛行和移动网络的不稳定性，使得渐进式渲染在先带前端开发中仍有用武之地。

【举例】

  • 图片懒加载————页面上的图片不会一次性全部加载。当用户滚动页面到图片部分时，JavaScript将加载并显示图像。
  • 分层次渲染(确定显示内容的优先级)————为了尽快将页面呈现给用户，页面只包含基本的最少量的CSS、脚本和内容。
    然后可以使用 延迟加载脚本 或监听DOMContentLoaded/ load事件加载其他资源和内容。
  • 异步加载HTML片段————当页面通过后台渲染时，把html拆分，通过异步请求，分块发送给浏览器。更多

Viewport：meta标签属性，字面意思为视图窗口，在移动web开发中使用。表示将设备浏览器宽度虚拟成一个特定的值(或计算得出)，这样利于移动web站点跨设备显示效果基本一致。
手机浏览器是把页面放在一个虚拟的“窗口”（viewport）中，通常这个虚拟的“窗口”（viewport）比屏幕宽，这样就不用把每个网页挤 到很小的窗口中（这样会破坏没有针对手机浏览器优化的网页的布局），用户可以通过平移和缩放来看网页的不同部分。

【举例】

iPhone4的屏幕大小是320像素*480像素，假设在浏览器中，320像素的屏幕宽度能够展示980像素宽度的内容。
那么320像素为可见视口，980像素为视窗视口的宽度。

手机浏览器基本支持。

为了显示更多内容，大多数浏览器会把自己的视窗视口扩大。即让原本320像素的屏幕宽度能够容下980像素的宽度甚至更宽(通过等比例缩小)。

【Viewport属性值】

  • width：设置layout viewport的宽度，为正整数/字符串"device-width"
  • height：设置layout viewport的宽度，为正整数/字符串"device-height"。不重要，很少使用。
  • initial-scale：设置页面的初始缩放比例，为一个可带小数的数字。当页面第一次load时缩放比例。
  • minimum-scale：允许用户的最小缩放值，为一个可带小数的数字。
  • maximum-scale：允许用户的最大缩放值，为一个可带小数的数字。
  • user-scalable：是否允许用户进行缩放，值为"yes"或"no"
这些属性可以同时使用，也可以单独使用或混合使用，多个属性同时使用时用逗号隔开。

常用针对移动网页优化过的页面的 viewport meta 标签：

<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1" >

【Reflow】

当涉及到DOM节点的布局属性发生变化时，就会重新计算该属性，浏览器会重新描绘相应的元素，此过程叫Reflow(回流或重排)

【Repaint】

当影响DOM元素可见性的属性发生变化(如color)时，浏览器会重新描绘相应的元素，此过程成为Repaint(重绘)。因此重排必然引起重绘。

【引起Reflow和Repaint的一些操作】

  • 调整窗口大小
  • 字体大小
  • 样式表变动
  • 元素内容变化，尤其是输入控件
  • CSS伪类激活，在用户交互过程中发生
  • DOM操作，DOM元素增删、修改
  • width, clientWidth scrollTop 等布局宽高的计算

【Repaint和Reflow是不可避免的，但应当使其对性能的影响减到最小】

  • 避免逐条更改样式，采用集中修改样式，如操作className
  • 避免频繁操作DOM。
    创建一个documentFragment 或div，在它上面应用所有的DOM操作，最后添加到文档里。设置在display:none的元素上操作，最后显示出来。
  • 避免频繁读取元素几何属性(如scrollTop)。绝对定位具有复杂动画的元素。
  • 绝对定位可以使元素脱离文档流，避免引起父元素及后续元素大量的回流。

【alt(属于img)】

如果无法显示图像，浏览器将显示alt指定的内容。

在鼠标移到元素上时显示title的内容。

【href(link,a...)】

href标识超文本引用，用在link和a等元素上。
href是引用和页面关联，是在当前元素和引用资源之间建立联系。

如果在文档中添加href，浏览器会识别该文档为css文件，就会并行下载资源并且不会停止对当前文档的处理。这也是为什么建议使用link而不是@import方式加载css的原因。

【src(img，script，iframe...)】

src表示引用资源，替换当前元素，用在img，script，iframe上。
src是页面内容不可缺少的一部分。
当浏览器解析到src，会暂停其他资源的下载和处理(图片不糊暂停其他资源下载)，直到把该资源加载、编译、执行完毕，类似于将所指向资源应用到当前内容。这也是为什么建议把js监本放在底部而不是头部的原因。

【为什么会出现浏览器兼容问题】

  • 同一产品，版本越老，bug越多
  • 同一产品，版本越新，功能越多
  • 不同产品，不同标准，不同实现方式

【处理兼容问题的思路】

  1. 要不要做：
    • 产品的角度：产品受众、受众的浏览器比例、效果优先还是基本功能优先
    • 成本的角度
  2. 做到什么程度：让哪些浏览器支持哪些效果
  3. 如何做：
    根据兼容需求选择技术框架/库(jquery)，兼容工具( html5shiv.js、 respond.js、css reset、normalize.css、Modernizr)* 条件注释、 CSS Hack、 js能力检测 做一些修补
    • 渐进增强(progressive enhancement)：针对低版本浏览器进行构建叶念，保证最基本的功能，然后再针对高级浏览器进行效果、交互等改进和追加功能达到更好的用户体验
    • 优雅降级(graceful degradation)：一开始就构建完整的功能，然后再针对低版本浏览器进行兼容。

doctype是一种标准通用标记语言的文档类型声明。目的是告诉标准通用标记语言解析器要使用什么样的文档类型定义(DTD)来解析文档。
声明是用来指示web浏览器关于页面使用哪个HTML版本进行编写的指令。声明必须是HTML文档的第一行，位于html标签之前。

浏览器本身分为两种模式，一种是标准模式，一种是怪异模式。浏览器通过doctype来区分这两种模式，doctype在html中的作用就是触发浏览器的标准模式，如果省略，浏览器就会进入到Quirks模式的怪异状态。在这种模式下，有些样式会和标准模式存在差异，而html标准和dom标准值规定了标准模式下的行为，没有对怪异模式做出规定，一次不同浏览器在鬼阿姨模式下的处理也是不同的，所以一定要在html开头使用doctype。

【行内元素】

一个行内元素只占据它对应标签的边框所包含的空间，一般情况下，行内元素只能包含数据和其他行内元素。

  • b, big, i, small, tt
  • abbr, acronym, cite, code, dfn, em, kdb, strong, samp, var
  • a, dbo, br, img, map, object, q, script, span, sub, sup
  • button, input, label, selcet, textarea

【块级元素】

占据一整行，高度、行高、内边距和外边距都可以改变，可以容纳块级标签和其他行内标签

  • header, form, ul, ol, table, article, div, hr, aside, figure, canvas, video, audio, footer

用于在网页内显示网页。

【优点】

  • iframe能够原封不动地把嵌入的网页展示出来。
  • 如果有多个网页引用iframe，那么你只需要修改iframe的内容，就可以实现调用的每一个页面内容的更改，方便快捷。
  • 网页如果为了统一风格，头部和版本都是一样的，就可以写成一个页面，用iframe来嵌套，可以增加代码的可重用。
  • 如果遇到加载缓慢的第三方内容如图标和广告，这些问题可以由iframe来解决。

【缺点】

  • 搜索引擎的麻涌程序无法解读这种页面
  • 框架结构中出现各种滚动条
  • 使用框架结构时，保证设置正确的导航链接
  • iframe页面会增加服务器的http请求

默认情况下，浏览器会记录用户网页上提交的输入框的信息。这样浏览器便能够做到自动完成（在用户开始输入的时候给用户提供可能的内容）和自动填充（在加载的时候预先填充某些字段）功能。
这些功能可能涉及用户的隐私，因此浏览器让用户可以禁用它（自动完成或自动填充），（不幸的是）它们默认是启用的。当然，许多提交的表单信息并不见得是有用的（例如：一次性的身份验证码-我把它理解为验证码）或是某些敏感信息（例如政府分发的身份识别码和信用卡安全码-我把它理解为身份证号和信用卡安全码）。即使浏览器的自动填充出去开启状态，网站（提供商）可能也不是那么喜欢浏览器记住这些字段的值。
如果想要关闭，那么就将不想要自动完成的form或input设置为autocomplete=off

【DOM】

Document Object Model，文档对象模型
DOM是为了操作文档出现的API，document是它的一个对象
DOM和文档有关，这里的文档指的是网页，也就是html文档。DOM和浏览器五官，他关注的事网页本身的内容。

【BOM】

Browser Object Model，浏览器对象模型
BOM是为了操作浏览器出现的API，window是它的一个对象
window对象即为JavaScript访问浏览器提供API，同时在ECMAScript中充当Global对象
