浏览器通过优先级规则，判断元素展示哪些样式。优先级通过4个维度指标确定，我们假定以a、b、c、d命名，分别代表以下含义：
  • a：表示是否使用内联样式(inline style)。如果使用，a为 1 ，否则为 0 。
  • b：表示id选择器的数量。
  • c：表示类选择器、 属性选择器 和伪类选择器数量之和。
  • d：表示标签(类型)选择器和 伪元素选择器 之和。

优先级的结果由每个值分开比较，abcd权重从左到右依次减小。判断优先级时，从左到右，一一比较，直到比较出最大值，即可停止。
所以，如果b的值不同，那么cd不会对结果产生影响。[eg：0,1,0,0比0,0,10,10的优先级高]
当出现优先级相等的情况时，最晚出现的样式规则会被采纳。如果样式表写了同样的规则(无论是在该文件内部还是其他样式文件中)，最后出现的(文件底部的)样式优先级更高，因此会被采纳。
在写样式时，应使用较低的优先级，这样这些样式可以被轻易地覆盖掉。尤其对写UI组件的时候更为重要，这样使用者就无需通过非常复杂的优先级规则或使用 !important 的方式，去覆盖组件的样式了。

【重置(Resetting)】

重置意味着除去所有的浏览器默认样式，例如使h1与p看起来完全一致，我们必须自己重新定义各种元素的样式。
如何重置：如border: none;

【标准化(normalizing)】

标准化没有去掉所有的默认样式，而是保留了有用的一部分，同时纠正了一些常见错误。 它使内置浏览器样式在各浏览器之间保持一致。

浮动(float)是CSS定位属性。浮动元素从网页的正常流动(文档流)中移出，但保持了部分的流动性， 块级元素对它视而不见，但内联元素会围绕它排列。这与绝对定位的元素不同，绝对定位会使元素完全从文档流中脱离。
使用css的clear属性(left，right，both)，将会让元素不允许身边存在浮动元素，移动到浮动元素下面。
如果父元素只包含浮动元素，那么该父元素的高度将坍塌为0。 可以通过clear:both从浮动元素后到父元素关闭前间一个空div来修复。 也可以设置浮动元素为一个 .clearfix类，再利用伪元素选择器::after清除浮动。

【代码】

.clearfix::after{
content: '';
display: block;
clear: both;
}

另外，如果父元素属性设置为 overflow: auto 或 overflow:hidden，会使其内部的子元素形成格式化上下文 (BFC )，并且父元素会扩张自己，使其能够包围它的子元素。

【z-index】

CSS中的z-index属性控制重叠元素的垂直叠加顺序。z-index只能影响position值不是static(默认)的元素。
没有定义z-index的值时，元素按照它们出现在DOM中的顺序堆叠(层级越低，位置越靠上)。非静态定位的元素(及其子元素)将始终覆盖静态定位(static)的元素，不论HTML层级结构。
【层叠上下文】

层叠上下文是包含一组图层的元素。在一组层叠上下文中， 其子元素的z-index值是相对于父元素而不是document root设置的。 每个层叠上下文完全独立于他的兄弟元素。如果元素B位于元素A之上， 那么即使A的子元素C具有比B更高的z-index， C也永远不会在B之上。
每个层叠上下文是自包含的：当元素的内容发生层叠后，整个该元素将会在 父层叠上下文 中按顺序进行层叠。少数css属性会触发一个新的层叠上下文。 如：opacity(透明度)小于1，transform(旋转变换)不是none，filter(滤镜，定义模糊，饱和度效果)不是none。

快格式上下文(BFC)是web页面的可视化css渲染的部分，是块级盒布局发生的区域，也是浮动元素和其他元素交互的区域。
一个html盒(box)满足以下任意一条，会创建块格式化上下文：

  • float的值不是none
  • position的值不是static或者relative
  • display的值是table-cell、table-caption、inline-block、 flex、或者inline-flex (表格，行内块，flex)
  • overflow(内容溢出如何处理)的值不是visible(内容出现在框外)
在BFC中，每个盒的左外边缘都与包含它的块的左边缘相接(如下图，左对齐【排列方式如果为右至左则右对齐)。

两个相邻的块集合在垂直方向删的边距会发生合并(collapse)。

雪碧图是把多张图片整合到一张上的图片，它被运用在众多使用了很多小图标的网站上(Gmail在使用)。实现方法：

  1. 使用生成器将多张图片打包成一张雪碧图，并为其生成合适的CSS。
  2. 每张图片都有相应的CSS类，该类定义了background-image、background-position和background-size属性
  3. 使用图片时，将相应的类添加到元素中。

【好处】

  • 减少加载多张图片的http请求数(一张雪碧图只需要一个请求)。但是对于 http2 而言，加载多张图片不是问题。
  • 提前加载资源，放置在需要时才在开始下载引发的问题，如只出现在:hover伪类中的图片，不会出现闪烁。

    一开始针对低版本的浏览器构建页面，满足最基本功能。 再针对高级浏览器进行效果交互、追加功能以达到更好的用户体验。

    • 【css写法】

    .transition { /*渐进增强写法*/
    -webkit-border-radius:30px 10px;
    -moz-border-radius:30px 10px;
    border-radius:30px 10px;
    }

    【额外知识】

    渐进增强和优雅降级是在CSS3出现之后才火起来的。 由于低级浏览器，比如IE6等，不支持CSS3的浏览器，但是CSS3的样式又特别优秀不忍放弃 ， 所以在高级浏览器中应用CSS3样式，在低级浏览器只保证基本功能。
    采用优雅降级的写法，如果一个浏览器同时支持前缀写法和正常写法， 后面的旧版浏览器样式就覆盖了新版样式，出现一些奇怪的问题 ， 但是用渐进增强的写法就不存在这个问题。这种属性不止border-radius一个， 所以为了避免这个不必要的错误，建议采用渐进增强的写法。 同时，也要看具体业务需求。如果受众单一，就采用优雅降级。

    问题1：大网站一般采取什么方式

【@media】

使用 @media 查询，你可以针对不同的媒体类型定义不同的样式。
@media 可以针对不同的屏幕尺寸设置不同的样式，特别是如果你需要设置设计响应式的页面，@media 是非常有用的。
当你重置浏览器大小的过程中，页面也会根据浏览器的宽度和高度重新渲染页面。

【值】

  • all：用于所有设备
  • print：用于打印机和打印预览
  • screen：用于电脑屏幕，平板，手机等
  • speech：用于屏幕阅读器等发声设备
【例子】

@media screen and (max-width: 300px) {
body {
background-color:lightblue;
}
}
如果文档宽度小于300像素则改变背景颜色为淡蓝。

首先，浏览器从关键选择器(最右边的选择器——key selector)，向左依次匹配。 根据关键选择器，浏览器从DOM中筛选出元素，然后向上遍历被选元素的父元素，判断是否匹配。

【例子】

如，对于 p span 的选择器，浏览器首先找到所有的span元素，并遍历它的父元素知道根元素找到p元素。然后才找到这个span元素。
选择器匹配语句链越短，那么浏览器的匹配速度就越快。 因此避免使用标签和通用选择器作为关键选择器，因为它们会匹配大量的元素，浏览器要进行大量工作去判断元素的父元素是否匹配。

BEM 原则上建议为独立的css类命名，并在需要层级关系时，将关系也体现在命名中，会使选择器高效且易于覆盖。
